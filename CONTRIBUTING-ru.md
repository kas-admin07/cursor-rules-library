# Вклад в Cursor Rules Library

Мы приветствуем ваш вклад! Пожалуйста, следуйте этим шагам для добавления или обновления правил:

## Шаги
1. Форкните репозиторий и создайте новую ветку для изменений.
2. Добавьте или обновите файлы правил `.mdc` в папке `.cursor/rules/`.
3. Для каждого файла `rule-name.mdc` добавьте перевод на русском как `rule-name-ru.mdc` с идентичной структурой.
4. Проверьте, что оба файла имеют одинаковые поля `description`, `globs` и `alwaysApply`.
5. Сделайте коммит с понятным сообщением, например `Add testing-strategies rule (EN & RU)`.
6. Откройте Pull Request в `main`.
7. В описании PR укажите:
   - Краткое описание изменений
   - Затронутые модули (core, frontend, backend и т.д.)
   - Конфликты или удалённые устаревшие правила

## Процесс ревью
- Ответственные проверят:
  - Корректное именование файлов (`*.mdc` и `*-ru.mdc`)
  - Согласованность англоязычного и русскоязычного контента
  - Правильность полей frontmatter
  - Наличие примеров valid/invalid
- PR должны проходить CI-проверки (линтер, проверка Markdown).

## Двуязычные файлы правил
- `example-rule.mdc` — английская версия
- `example-rule-ru.mdc` — русская версия

Убедитесь, что версии синхронизированы.

## Формат файлов правил
Каждый файл правил должен следовать этой структуре:

description: "Подробное описание когда применять это правило"
globs: *.tsx, *.ts (или оставить пустым)
alwaysApply: true/false

Название правила
Критические правила
Выполнимое правило 1

Выполнимое правило 2

Выполнимое правило 3

Примеры
<example> ✅ Правильная реализация </example> <example type="invalid"> ❌ Неправильная реализация </example> ```
